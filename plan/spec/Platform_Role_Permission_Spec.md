## 1. 권한 제어 핵심 매커니즘 (3-Step Filter)

플랫폼 내의 모든 데이터(슬롯 및 결과물) 접근은 아래 3단계 필터를 **순차적으로** 거쳐 검증됩니다.
하나의 필터라도 통과하지 못하면 데이터 접근 및 노출이 **원천 차단**됩니다.

```
┌─────────────────────────────────────────────────────────────┐
│  1단계: 광고주 권한 키 (Key)                                  │
│  → 유저가 해당 슬롯에 매핑된 '광고주' 권한을 보유하고 있는가?  │
├─────────────────────────────────────────────────────────────┤
│  2단계: 슬롯 공개 범위 필터                                    │
│  → 해당 슬롯이 어떤 공개 수준으로 세팅되어 있는가?             │
├─────────────────────────────────────────────────────────────┤
│  3단계: 유저 롤(Role) 매칭                                    │
│  → 유저의 롤이 슬롯의 공개 수준에서 허용하는 롤인가?           │
└─────────────────────────────────────────────────────────────┘
```

### 1.1 필터 순서도

```
요청 → [1단계] 광고주 권한 보유? 
         ├─ NO → ❌ 접근 차단
         └─ YES → [2단계] 슬롯 공개 범위 확인
                    ├─ Private → [생성자 본인?]
                    │              ├─ NO → ❌ 접근 차단
                    │              └─ YES → ✅ 접근 허용
                    ├─ Internal → [3단계] 롤 = Marketer?
                    │              ├─ NO → ❌ 접근 차단
                    │              └─ YES → ✅ 접근 허용
                    └─ Shared → [3단계] 롤이 공유 대상에 포함?
                                 ├─ NO → ❌ 접근 차단
                                 └─ YES → ✅ 접근 허용
```

## 2. 유저 롤(Role) 정의

플랫폼을 이용하는 유저는 비즈니스 목적과 소속에 따라 다음 **3가지 롤**로 분류됩니다.

| 롤 | 영문 | 정의 | 권한 수준 |
|---|---|---|---|
| 마케터 | Marketer | 플랫폼 내부 운영 및 광고주 관리를 담당하는 마스터 유저. 모든 슬롯 권한의 기본 통제권을 가짐 | 최상위 |
| 클라이언트 | Client | 광고주 본사 담당자 롤. 마케터가 공유(Shared)한 자산에 접근 가능 | 제한적 |
| 에이전시 | Agency | 광고 대행사 담당자 롤. 마케터가 공유(Shared)한 자산 중 대행사 대상 자산에 접근 가능 | 제한적 |

### 2.1 롤별 기본 권한 요약

| 기능 | Marketer | Client | Agency |
|------|----------|--------|--------|
| 슬롯 생성 | ✅ | ❌ | ❌ |
| 슬롯 수정/삭제 | ✅ (본인 생성분) | ❌ | ❌ |
| 슬롯 공개 범위 설정 | ✅ | ❌ | ❌ |
| Private 슬롯 접근 | ✅ (본인만) | ❌ | ❌ |
| Internal 슬롯 접근 | ✅ | ❌ | ❌ |
| Shared 슬롯 접근 | ✅ | ✅ (대상 지정 시) | ✅ (대상 지정 시) |
| 시나리오 생성 | ✅ | ❌ | ❌ |
| 시나리오 결과 조회 | ✅ | ✅ (Shared 슬롯 내) | ✅ (Shared 슬롯 내) |
| DataShot 데이터셋 생성 | ✅ | ❌ | ❌ |
| DataShot 결과 조회 | ✅ | ✅ (Shared 슬롯 내) | ✅ (Shared 슬롯 내) |
| SpinX 사용 | ✅ | ✅ | ✅ |

## 3. 슬롯(Slot) 공개 범위 정의

슬롯은 아래와 같이 **3단계 수준**의 공개 범위를 가지며, 오른쪽으로 갈수록 접근 권한이 확장되는 구조입니다.

```
개인용 (Private) → 인터널 (Internal) → 쉐어드 (Shared)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  좁음 ◀─────────── 접근 범위 ───────────▶ 넓음
```

### 3.1 개인용 (Private)

| 항목 | 내용 |
|------|------|
| 정의 | 슬롯을 생성한 작업자 개인의 독립된 작업 공간 |
| 접근 정책 | 광고주 권한이나 유저 롤에 상관없이, **슬롯 생성자 본인 외에는 타인의 접근이 절대 불가** |
| 사용 시나리오 | 초안 작업, 개인 실험, 미공개 분석 |
| UI 표시 | 🔒 Private 뱃지 |

### 3.2 인터널 (Internal)

| 항목 | 내용 |
|------|------|
| 정의 | 해당 광고주를 담당하는 내부 마케터들끼리만 공유하는 보안 공간 |
| 접근 정책 | 해당 광고주 권한을 가진 유저 중, **[마케터] 롤을 가진 유저만** 접근 가능 (클라이언트, 에이전시 접근 불가) |
| 사용 시나리오 | 내부 전략 논의, 마케터 간 협업, 클라이언트 공유 전 검토 |
| UI 표시 | 🏢 Internal 뱃지 |

### 3.3 쉐어드 (Shared)

| 항목 | 내용 |
|------|------|
| 정의 | 외부 파트너(클라이언트 또는 에이전시)와 협업하기 위해 명시적으로 공유된 공간 |
| 접근 정책 | 마케터가 슬롯 설정 시 **공유 대상을 선택** |
| 사용 시나리오 | 클라이언트 리포팅, 대행사 협업, 외부 공유 |
| UI 표시 | 🌐 Shared 뱃지 + 공유 대상 표시 |

**Shared 공유 대상 옵션**:

| 공유 대상 선택 | 접근 허용 롤 | 차단 롤 |
|---|---|---|
| 에이전시 선택 | Marketer + Agency | Client |
| 클라이언트 선택 | Marketer + Client | Agency |
| 에이전시 + 클라이언트 선택 | Marketer + Agency + Client | — |

## 5. 데이터 노출 정책

### 5.1 SlotBoard (메인 대시보드)

- 유저에게는 **3-Step Filter를 통과한 슬롯만** 목록에 노출
- 접근 불가 슬롯은 목록 자체에서 제외 (존재 여부도 비노출)
- 슬롯 카드에 공개 범위 뱃지 표시 (Private / Internal / Shared)

### 5.2 SNB (Side Navigation Bar)

- 트리 구조에 노출되는 슬롯/결과물도 동일한 3-Step Filter 적용
- 접근 불가 항목은 트리에서 완전 제거

### 5.3 검색 결과

- 검색 시에도 3-Step Filter 적용
- 권한 없는 슬롯/결과물은 검색 결과에서 제외

### 5.4 URL 직접 접근

- 권한 없는 슬롯의 URL로 직접 접근 시도 시 → 403 에러 페이지 표시
- "접근 권한이 없습니다. 슬롯 관리자에게 문의하세요." 메시지

## 6. 슬롯 생성 시 공개 범위 설정 UI

### 6.1 설정 위치

- 슬롯 생성 다이얼로그 내 "공개 범위" 필드
- 슬롯 설정 > 공개 범위 변경

### 6.2 UI 구성

```
┌─────────────────────────────────────────┐
│ 공개 범위 *                              │
│                                         │
│ ○ Private (나만 접근 가능)               │
│ ○ Internal (내부 마케터만 접근 가능)      │
│ ○ Shared (외부 파트너와 공유)            │
│                                         │
│ [Shared 선택 시 추가 옵션]               │
│ ┌─────────────────────────────────────┐ │
│ │ 공유 대상 선택:                      │ │
│ │ ☐ 클라이언트 (광고주 본사)           │ │
│ │ ☐ 에이전시 (광고 대행사)             │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

### 6.3 기본값

- 슬롯 생성 시 기본 공개 범위: **Internal**

## 7. 권한 변경 시 영향

### 7.1 공개 범위 축소 시 (예: Shared → Internal)

- 기존에 접근 가능했던 Client/Agency 유저의 접근이 즉시 차단
- 해당 유저의 SlotBoard, SNB에서 슬롯 즉시 제거
- 진행 중이던 SpinX 세션은 유지되나, 새 질문 전송 시 권한 재검증

### 7.2 공개 범위 확장 시 (예: Internal → Shared)

- 새로 접근 가능해진 유저의 SlotBoard, SNB에 슬롯 즉시 노출
- 알림 발송 (선택적)

### 7.3 광고주 권한 회수 시

- 해당 광고주의 모든 슬롯 접근 즉시 차단
- SlotBoard에서 해당 광고주 슬롯 전체 제거
