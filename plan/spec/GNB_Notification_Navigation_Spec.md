# GNB 알림 센터 — 알림 클릭 네비게이션 정책 명세서

> 목적: QA 테스트케이스(TC) 작성 기준 문서  
> 버전: v1.0  
> 작성일: 2026-05-26  
> 범위: GNB 알림 센터에서 알림 클릭 시 랜딩 및 유효성 검증 로직

---

## 1. 개요

GNB 알림 센터의 알림 항목을 클릭했을 때, 대상 리소스(시나리오/데이터셋)의 유효성을 단계적으로 검증하고 적절한 페이지로 랜딩하는 정책을 정의한다.

---

## 2. GNB 알림 bar 노출 정책

### 2.1 알림 bar 위치

- GNB 상단 영역에 1줄 텍스트로 노출
- 알림 센터(벨 아이콘) 좌측 또는 GNB 중앙 영역

### 2.2 노출 조건

| 조건 | 노출 내용 |
|---|---|
| 알림 원천 0개 | 고정 메시지: "Ready to Set? 최상의 성과를 위한 전략을 세팅하세요." |
| 알림 원천 1개 이상 | 가장 최근 알림 메시지 노출 |

### 2.3 GNB bar 노출 메시지

| 타입 | 노출 메시지 | 예시 |
|---|---|---|
| 작업 알림 | $솔루션약자(/포함)+$resourceName+:+$title | R/C 테스트_RF(종편만): 시나리오 생성이 완료되었습니다.<br>D/S 모먼트 건강식품: 데이터셋 생성이 완료되었습니다. |
| 사용자 공지 | 공지: $title | 공지: 최신 데이터가 업데이트되었습니다. |
| 사용자 레벨 안내 | 레벨 안내: $title | 레벨 안내: {다음 등급명}까지 50% 도달했습니다.<br>레벨 안내: {달성 등급명} 레벨에 도달했습니다. |

### 2.4 메시지 변수 정의

| 변수 | 설명 | 예시 값 |
|---|---|---|
| `$솔루션약자` | 솔루션 축약 표기 | R/C (Reach Caster), D/S (DataShot) |
| `$resourceName` | 슬롯명 또는 리소스명 | 테스트_RF(종편만), 모먼트 건강식품 |
| `$title` | 알림 본문 메시지 | 시나리오 생성이 완료되었습니다 |

### 2.5 노출 규칙

| 항목 | 스펙 |
|---|---|
| 최대 노출 줄 수 | 1줄 (말줄임 처리) |
| 갱신 주기 | 새 알림 수신 시 즉시 갱신 |
| 클릭 동작 | 알림 센터 패널 열기 |
| 알림 0개 시 클릭 | 동작 없음 또는 알림 센터 빈 상태 노출 |

## 3. 시나리오 생성 완료 알림 클릭

### 3.1 검증 흐름 (3단계)

```
[알림 클릭]
    │
    ▼
[1단계] 슬롯 ID 일치 여부 검증
    │
    ├─ [YES] 슬롯 ID 일치 (기존 슬롯)
    │       │
    │       ▼
    │   [2단계] 상위 슬롯 유효성 검증
    │       │
    │       ├─ [NO] 슬롯 삭제/권한 없음 → SlotBoard 이동 + Error 토스트
    │       │
    │       └─ [YES] 슬롯 정상
    │               │
    │               ▼
    │           [3단계] 시나리오 유효성 검증
    │               │
    │               ├─ [NO] 시나리오 삭제됨 → 빈 상세 1~2초 → 목록 리다이렉트 + Error 토스트
    │               │
    │               └─ [YES] 시나리오 존재 → 시나리오 상세 페이지 이동
    │
    └─ [NO] 슬롯 ID 불일치 (다른 슬롯으로 이동됨)
            │
            ▼
        이동된 신규 슬롯 최종 검증
            │
            ├─ [정상] 신규 슬롯 권한 있음 + 시나리오 존재
            │       → 변경된 슬롯의 시나리오 상세 페이지 이동 + Info 토스트
            │
            ├─ [예외1] 신규 슬롯 삭제/권한 없음
            │       → SlotBoard 이동 + Error 토스트
            │
            └─ [예외2] 시나리오 최종 삭제됨
                    → 빈 상세 1~2초 → 해당 슬롯 시나리오 목록 리다이렉트 + Error 토스트
```

### 3.2 1단계: 슬롯 ID 일치 여부 검증

| 조건 | 설명 |
|---|---|
| 비교 대상 | 알림의 슬롯 ID (생성 시점) vs 시나리오의 현재 슬롯 ID |
| YES | 동일 슬롯에 위치 → 2단계 진입 |
| NO | 다른 슬롯으로 이동됨 → 이동된 슬롯 검증 |

### 3.3 2단계: 상위 슬롯(Slot) 유효성 검증

| 조건 | 결과 | 랜딩 | 토스트 |
|---|---|---|---|
| 슬롯 삭제됨 | 즉시 리다이렉트 | SlotBoard (`/slotboard`) | Error |
| 슬롯 권한 없음 | 즉시 리다이렉트 | SlotBoard (`/slotboard`) | Error |
| 슬롯 정상 | 3단계 진입 | - | - |

| 토스트 상세 |  |
|---|---|
| 타입 | Error |
| 타이틀 | 접근 불가 항목 |
| 내용 | "Slot이 삭제되었거나 접근 권한이 없습니다." |

### 3.4 3단계: 시나리오 유효성 검증

| 조건 | 결과 | 랜딩 | 토스트 |
|---|---|---|---|
| 시나리오 삭제됨 | 빈 상세 1~2초 노출 후 리다이렉트 | 해당 슬롯의 시나리오 목록 페이지 | Error |
| 시나리오 존재 | 정상 이동 | 시나리오 상세 페이지 | 없음 |

| 토스트 상세 (삭제됨) |  |
|---|---|
| 타입 | Error |
| 타이틀 | 존재하지 않는 항목 |
| 내용 | "삭제되었거나 존재하지 않는 시나리오입니다." |

### 3.5 슬롯 이동 케이스 (1단계 NO)

#### 정상: 이동한 슬롯 권한 있음 + 시나리오 존재

| 항목 | 스펙 |
|---|---|
| 랜딩 | 현재 변경된 슬롯 내의 시나리오 상세 페이지 |
| 브레드크럼 주의 | 상단 브레드크럼 슬롯명에 '이전 슬롯명'이 1초 정도 노출될 수 있음 |
| 토스트 타입 | Info |
| 토스트 타이틀 | Slot 이동 안내 |
| 토스트 내용 | "'{현재 Slot명}' Slot에 위치한 시나리오 상세 페이지입니다." |

#### 예외1: 이동한 신규 슬롯 삭제/권한 없음

| 항목 | 스펙 |
|---|---|
| 랜딩 | SlotBoard (`/slotboard`) |
| 토스트 타입 | Error |
| 토스트 타이틀 | 접근 불가 항목 |
| 토스트 내용 | "Slot이 삭제되었거나 접근 권한이 없습니다." |

#### 예외2: 이동 후 시나리오 최종 삭제됨

| 항목 | 스펙 |
|---|---|
| 랜딩 (1차) | 데이터가 비어있는 상세 페이지 1~2초 노출 |
| 랜딩 (2차) | 해당 슬롯 하위의 시나리오 목록 페이지로 강제 리다이렉트 |
| 토스트 타입 | Error |
| 토스트 타이틀 | 존재하지 않는 항목 |
| 토스트 내용 | "삭제되었거나 존재하지 않는 시나리오입니다." |

## 4. 시나리오 생성 실패 알림 클릭

### 4.1 검증 흐름 (1단계)

```
[알림 클릭]
    │
    ▼
[1단계] 상위 슬롯(Slot) 유효성 검증
    │
    ├─ [NO] 슬롯 삭제/권한 없음
    │       → SlotBoard 이동 + Error 토스트
    │
    └─ [YES] 슬롯 정상
            → 해당 슬롯 하위의 시나리오 목록 페이지로 이동
```

### 4.2 상세

| 조건 | 결과 | 랜딩 | 토스트 |
|---|---|---|---|
| 슬롯 삭제/권한 없음 | 즉시 리다이렉트 | SlotBoard (`/slotboard`) | Error |
| 슬롯 정상 | 정상 이동 | 해당 슬롯의 시나리오 목록 페이지 | 없음 |

| 토스트 상세 (슬롯 무효) |  |
|---|---|
| 타입 | Error |
| 타이틀 | 접근 불가 항목 |
| 내용 | "Slot이 삭제되었거나 접근 권한이 없습니다." |

## 5. URL 직접 접근 시 검증 정책

알림 클릭이 아닌, 사용자가 URL을 직접 입력하여 접근하는 경우에도 동일한 검증 로직을 적용한다.

### 5.1 결과물이 삭제됐거나 없는 ID로 접근

| 항목 | 스펙 |
|---|---|
| 랜딩 (1차) | 데이터가 비어있는 상세 페이지 1~2초 노출 |
| 랜딩 (2차) | 해당 슬롯 하위의 시나리오(또는 데이터셋) 목록 페이지로 이동 |
| 예외 | 슬롯마저 무효한 경우 → 4.3 규칙 적용 |
| 토스트 타입 | Error |
| 토스트 타이틀 | 존재하지 않는 항목 |
| 토스트 내용 | "삭제되었거나 존재하지 않는 시나리오입니다." (또는 "...데이터셋입니다.") |

### 5.2 결과물이 다른 슬롯으로 이동된 상태에서 이전 URL로 접근

| 항목 | 스펙 |
|---|---|
| 랜딩 | 현재 변경된 새로운 슬롯의 상세 페이지로 이동 |
| 토스트 타입 | Info |
| 토스트 타이틀 | Slot 이동 안내 |
| 토스트 내용 | "'{현재 Slot명}' Slot에 위치한 시나리오 상세 페이지입니다." |

### 5.3 슬롯이 삭제됐거나 권한 없는 슬롯 ID로 접근

| 항목 | 스펙 |
|---|---|
| 랜딩 | SlotBoard (`/slotboard`) |
| 토스트 타입 | Error |
| 토스트 타이틀 | 접근 불가 항목 |
| 토스트 내용 | "Slot이 삭제되었거나 접근 권한이 없습니다." |

```
    │       → SlotBoard 이동 + Error 토스트
    │
    └─ [YES] 슬롯 정상
            → 해당 슬롯 하위의 데이터셋 목록 페이지 (`/datashot`)로 이동
```

### 7.2 상세

| 조건 | 결과 | 랜딩 | 토스트 |
|---|---|---|---|
| 슬롯 삭제/권한 없음 | 즉시 리다이렉트 | SlotBoard (`/slotboard`) | Error |
| 슬롯 정상 | 정상 이동 | 해당 슬롯의 데이터셋 목록 페이지 | 없음 |

| 토스트 상세 (슬롯 무효) |  |
|---|---|
| 타입 | Error |
| 타이틀 | 접근 불가 항목 |
| 내용 | "Slot이 삭제되었거나 접근 권한이 없습니다." |
