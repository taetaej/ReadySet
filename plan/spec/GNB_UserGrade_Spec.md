# GNB 사용자 등급 시스템 — 화면 상세 명세서

> 목적: QA 테스트케이스(TC) 작성 기준 문서  
> 버전: v1.0  
> 작성일: 2026-05-26  
> 범위: GNB 사용자 등급 표시 및 hover 카드 UI, 승급/이정표 알림 정책

---

## 1. 개요

GNB 우측에 현재 사용자 등급을 표시하고, 클릭 시 등급 상세 카드(hover 카드)를 노출하여 현재 레벨, 승급 조건 진행률, 솔루션 커버리지를 안내한다.

---

## 2. 등급 체계

### 2.1 전체 6단계 등급

| 순서 | 등급명 | 아이콘 | 설명 |
|---|---|---|---|
| 1 | Slot-In Ready | Zap | ReadySet의 잠재력을 탐색 중인 예비 전략가. |
| 2 | Active Slotter | Activity | 데이터의 흐름을 만들기 시작한 실무 전략가. |
| 3 | Strategy Builder | Target | 개별 솔루션을 조합해 전략의 기틀을 잡는 숙련가. |
| 4 | Solution Expert | Award | 플랫폼을 완벽히 활용해 최적의 해답을 도출하는 전문가. |
| 5 | Master Architect | Crown | 복잡한 시나리오를 구조화하여 전략 생태계를 구축한 마스터. |
| 6 | ReadySet Visionary | Star | 플랫폼의 한계를 넘어 전략의 새 지평을 여는 독보적 선구자. |

### 2.2 승급 조건

| 조건 | 설명 |
|---|---|
| 보유 결과물 수 | 현재 총 보유량 ≥ 다음 등급 기준 |
| 솔루션 커버리지 | 사용한 솔루션 수 ≥ 다음 등급 기준 |
| 결합 조건 | 보유 결과물 수 AND 솔루션 커버리지 모두 충족 시 승급 |

---

## 3. GNB 등급 표시 (트리거)

| 요소 | 스펙 |
|---|---|
| 위치 | GNB 우측 영역 |
| 표시 | 현재 등급명 텍스트 (font-size: 13px, font-weight: 500) |
| 색상 | primary-foreground |
| 클릭 | 등급 카드 토글 (열기/닫기) |
| 스타일 | padding: 6px 12px, border-radius: 16px |

---

## 4. 등급 카드 (Hover Card UI)

### 4.1 카드 기본 스펙

| 요소 | 스펙 |
|---|---|
| 위치 | position: absolute, top: 100%, right: 0 |
| 크기 | width: 280px |
| 스타일 | bg: card, border: border/0.6, border-radius: 16px, box-shadow |
| 열기 | 등급명 클릭 |
| 닫기 | 외부 클릭 (mousedown) |
| 이벤트 전파 | 카드 내부 클릭 시 stopPropagation |

### 4.2 구성 요소

#### 4.2.1 레벨 정보 영역

| 요소 | 스펙 |
|---|---|
| 현재 레벨명 | font-size: 15px, font-weight: 700 |
| 설명 | font-size: 10px, muted-foreground |

#### 4.2.2 전체 계단식 6레벨 표기

| 요소 | 스펙 |
|---|---|
| 레이아웃 | 6개 바 가로 배치 (flex, 각 flex: 1) |
| 바 높이 | 20px + (index × 6px) — 계단식 증가 |
| 현재 등급 바 | bg: primary |
| 기타 등급 바 | bg: muted |
| 라벨 | font-size: 6px, 현재 등급: primary + bold, 기타: muted-foreground |

#### 4.2.3 승급 정보 — "Let's Level Up!"

| 요소 | 스펙 |
|---|---|
| 타이틀 | "Let's Level Up!" (font-size: 15px, font-weight: 700) |
| 구분선 | border-top: 1px solid border/0.4, padding-top: 16px |

#### 4.2.4 Mission 1: 보유 결과물 수

| 요소 | 스펙 |
|---|---|
| 라벨 | "Mission 1" (font-size: 8px) + "결과물 {N}개 보유하기" (font-size: 10px) |
| 진행률 표시 | `{현재}/{목표}` (font-size: 8px, primary 색상, 우측 정렬) |
| 프로그레스 바 | height: 4px, border-radius: 2px, bg: muted, fill: primary |
| 마일스톤 | 25%, 50%, 75% 위치에 1px 구분선 |
| 애니메이션 | width transition 0.6s cubic-bezier |
| 동적 메시지 | 진행률에 따라 변경 (아래 참조) |

#### 4.2.5 프로그레스 바 계산식

```
진행률(%) = MAX(0, (현재 총 보유량 - 현재 등급 하한선) / (다음 등급 목표치 - 현재 등급 하한선)) * 100
```

- 결과물 수 100% 초과했으나 솔루션 커버리지 미달성 시 프로그레스 바 100%로 캡핑

#### 4.2.6 진행률별 동적 메시지

| 구간 | 메시지 |
|---|---|
| 0% | "다음 등급을 향한 여정을 시작하세요." |
| 0% 초과 ~ 25% 미만 | "좋은 출발이에요!" |
| 25% 이상 ~ 50% 미만 | "순조롭게 진행중이네요!" |
| 50% 이상 ~ 75% 미만 | "절반 이상 왔어요!" |
| 75% 이상 ~ 100% 미만 | "조금만 더 힘을 내요!" |
| 100% | "필요한 결과물을 다 채웠어요!" |

#### 4.2.7 Mission 2: 솔루션 커버리지

| 요소 | 스펙 |
|---|---|
| 라벨 | "Mission 2" (font-size: 8px) + "솔루션 {N}개 이상 사용하기" (font-size: 10px) |
| 완료 아이콘 | CheckCheck (14px), 완료 시 stampPop 애니메이션 |
| 솔루션 목록 | 4개 원형 아이콘 가로 배치 |

#### 4.2.8 솔루션 커버리지 목록

| 솔루션 | 색상 | 상태 |
|---|---|---|
| DataShot | #7B61FF | 활성 (done: true) |
| Reach Caster | #00C8FF | 미완료 (done: false) |
| Ad Curator | #00E676 | 잠금 (locked: true, "준비중") |
| Budget Optimizer | #FF3B7A | 잠금 (locked: true, "준비중") |

| 상태 | UI |
|---|---|
| done: true | 색상 배경(15% opacity) + Flame 아이콘 (fill) |
| done: false, locked: false | 점선 border + Flame 아이콘 (outline) |
| locked: true | 점선 border + "준비중" 텍스트 |

## 5. 알림 발송 정책

### 5.1 알림 발송 케이스

| Case | 알림 종류 |
|---|---|
| 승급 조건 충족 (보유 결과물 수 ≥ 다음 등급 기준 AND 솔루션 커버리지 충족) | 승급 알림 발송 (+ 솔루션 커버리지 초기화) |
| 이정표 도달 (25/50/75%) | 이정표 알림 발송 |
| 승급 + 이정표 동시 도달 | Only 승급 알림 발송 (이정표 알림 생략) |
| 이전에 발송한 이정표 재도달 | 중복 발송함 |

### 5.2 승급 알림

| 항목 | 스펙 |
|---|---|
| 트리거 | 보유 결과물 수 AND 솔루션 커버리지 모두 충족 |
| 부수 효과 | 솔루션 커버리지 초기화 (새 등급 기준으로 리셋) |
| 알림 타입 | GNB 알림 센터에 등록 |

### 5.3 이정표 알림

| 항목 | 스펙 |
|---|---|
| 트리거 | 프로그레스 바 25%, 50%, 75% 도달 시점 |
| 중복 발송 | 허용 (이전에 발송한 이정표 재도달 시에도 발송) |
| 승급과 동시 | 승급 알림만 발송, 이정표 알림 생략 |

## 6. 에러 처리 및 엣지 케이스

| 시나리오 | 동작 |
|---|---|
| 등급 데이터 로딩 실패 | 등급명 영역 빈 상태 또는 이전 캐시 표시 |
| 최고 등급 도달 (ReadySet Visionary) | "Let's Level Up!" 영역 대신 축하 메시지 또는 미표시 |
| 솔루션 커버리지 잠금 상태 | "준비중" 표시, 클릭 불가 |
| 카드 외부 클릭 | 카드 닫힘 |

---

## 7. 접근성 고려사항

| 항목 | 기준 |
|---|---|
| 등급 버튼 | role="button", aria-expanded, aria-haspopup |
| 카드 | role="dialog" 또는 role="tooltip" |
| 프로그레스 바 | role="progressbar", aria-valuenow, aria-valuemin, aria-valuemax |
| 키보드 | Enter/Space로 카드 토글, ESC로 닫기 |
| 색상 대비 | 아이콘 + 텍스트 병행 (색상만으로 상태 구분하지 않음) |
