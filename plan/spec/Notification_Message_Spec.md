## 1. 알림 유형

### 1.1 다이얼로그 (Dialog)
- **용도**: 사용자의 명시적 확인/취소가 필요한 중요한 액션
- **특징**: 모달 형태, 배경 딤 처리, 사용자 응답 필수
- **버튼**: 확인/취소, 예/아니오, 삭제/취소 등

### 1.2 토스트 (Toast)
- **용도**: 작업 완료, 성공/실패 등 일시적 피드백
- **특징**: 자동 사라짐 (3-5초), 화면 하단 또는 상단
- **타입**: Success, Error, Warning, Info

### 1.3 인라인 유효성 검사 (Inline Validation)
- **용도**: 입력 필드의 실시간 유효성 검사
- **특징**: 필드 하단에 에러 메시지 표시, 빨간색 텍스트
- **타이밍**: 포커스 아웃 또는 실시간

### 1.4 플레이스홀더/힌트 (Placeholder/Hint)
- **용도**: 입력 가이드, 예시 제공
- **특징**: 회색 텍스트, 입력 시 사라짐
- **위치**: 입력 필드 내부 또는 하단

### 1.5 빈 상태 메시지 (Empty State)
- **용도**: 데이터가 없을 때 안내
- **특징**: 중앙 정렬, 아이콘 + 텍스트
- **위치**: 콘텐츠 영역 중앙

| 파일 다운로드 실패 | Error | 파일 다운로드에 실패했습니다 |

### 7.6 권한 관련

**토스트**
| 상황 | 유형 | 메시지 |
|------|------|--------|
| 읽기 전용 모드 | Info | 읽기 전용 모드입니다<br>수정 권한이 없습니다 |
| 권한 없음 | Error | 이 작업을 수행할 권한이 없습니다 |
| 소유자만 가능 | Error | 소유자만 수정/삭제할 수 있습니다 |

**다이얼로그**
| 상황 | 제목 | 내용 | 버튼 |
|------|------|------|------|
| 권한 없음 | 권한이 없습니다 | 이 작업을 수행할 권한이 없습니다.<br>관리자에게 문의하세요. | 확인 |

## 8. 메시지 작성 가이드라인

### 8.1 톤 앤 매너
- **명확하고 간결하게**: 사용자가 즉시 이해할 수 있도록
- **친절하되 전문적으로**: 존댓말 사용, 기술 용어는 필요시에만
- **액션 지향적**: 사용자가 다음에 무엇을 해야 하는지 명확히

### 8.2 에러 메시지 원칙
1. **무엇이 잘못되었는지** 명확히 설명
2. **왜 잘못되었는지** 간단히 설명 (필요시)
3. **어떻게 해결할 수 있는지** 제시

**좋은 예시**:
- ❌ "오류가 발생했습니다"
- ✅ "네트워크 연결을 확인해주세요"

**나쁜 예시**:
- ❌ "Error: 500 Internal Server Error"
- ✅ "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요"

### 8.3 성공 메시지 원칙
- 간결하게 완료 사실만 전달
- 불필요한 축하 표현 지양
- 다음 액션이 있다면 함께 안내

**좋은 예시**:
- ✅ "Slot이 생성되었습니다"
- ✅ "시나리오 생성 요청이 완료되었습니다. 처리 완료 시 알림 센터에서 알려드립니다"

### 8.4 확인 다이얼로그 원칙
- 제목: 간결한 질문 형태
- 내용: 구체적인 설명 + 주의사항
- 버튼: 명확한 액션 표현

**좋은 예시**:
```
제목: "Slot을 삭제하시겠습니까?"
내용: "{Slot명}"을(를) 삭제합니다.
      Slot 내 {N}개 시나리오가 함께 삭제되며, 
      삭제된 데이터는 복구할 수 없습니다.
버튼: [삭제] [취소]
```

### 8.5 다국어 대응 고려사항
- 변수 처리: `{변수명}` 형태로 표기
- 문장 구조: 언어별 어순 차이 고려
- 문자 길이: 영문 기준 1.5배 공간 확보

## 9. 구현 참고사항

### 9.1 우선순위

**P0 (필수 - 즉시 구현)**
- Slot 생성/수정/삭제 관련 메시지
- 시나리오 생성/삭제 관련 메시지
- 필수 입력 필드 유효성 검사
- 네트워크 에러 메시지
- 세션 만료 메시지

**P1 (중요 - 1차 배포 전)**
- 시나리오 복제/이동 관련 메시지
- 검색/필터 결과 없음 메시지
- 데이터 로딩 상태 메시지
- 파일 다운로드 관련 메시지
- 권한 관련 메시지

**P2 (권장 - 2차 개선)**
- 자동 저장 메시지
- 페이지 이탈 경고
- 파일 업로드 관련 메시지
- 상세한 에러 메시지 (개발자 모드)

### 9.2 접근성 고려사항

**스크린 리더 지원**
- 에러 메시지는 `aria-live="assertive"` 속성 사용
- 성공 메시지는 `aria-live="polite"` 속성 사용
- 다이얼로그는 `role="alertdialog"` 속성 사용
- 토스트는 `role="status"` 또는 `role="alert"` 속성 사용

**시각적 표현**
- 색상만으로 에러를 표시하지 않고 아이콘/텍스트 병행
- 에러: 빨간색 + X 아이콘
- 성공: 초록색 + 체크 아이콘
- 경고: 노란색 + 느낌표 아이콘
- 정보: 파란색 + i 아이콘

**키보드 접근성**
- 다이얼로그는 Tab 키로 버튼 간 이동 가능
- Enter 키로 기본 액션 실행
- Escape 키로 다이얼로그 닫기
- 토스트는 포커스 받지 않음 (자동 사라짐)

### 9.3 테스트 체크리스트

**유효성 검사 테스트**
- [ ] 모든 필수 입력 필드 미입력 시 에러 메시지 표시
- [ ] 글자 수 제한 초과 시 에러 메시지 표시
- [ ] 숫자 필드에 문자 입력 시 에러 메시지 표시
- [ ] 비중 합계 100% 검증 (Ratio Finder)
- [ ] 날짜 범위 검증 (시작일 ≤ 종료일)
- [ ] 리치커브 구간 검증 (최소값 < 최대값, 구간 수 ≤ 20)

**다이얼로그 테스트**
- [ ] 삭제 확인 다이얼로그 표시
- [ ] 이동 확인 다이얼로그 표시
- [ ] 페이지 이탈 경고 다이얼로그 표시
- [ ] 확인 버튼 클릭 시 액션 실행
- [ ] 취소 버튼 클릭 시 다이얼로그 닫기
- [ ] Escape 키로 다이얼로그 닫기
- [ ] 배경 클릭 시 다이얼로그 닫기 (선택적)

**토스트 테스트**
- [ ] 성공 토스트 3-5초 후 자동 사라짐
- [ ] 에러 토스트 5초 후 자동 사라짐
- [ ] 여러 토스트 동시 표시 시 스택 형태로 표시
- [ ] X 버튼 클릭 시 즉시 사라짐
- [ ] 화면 크기에 따라 위치 조정

**네트워크 에러 테스트**
- [ ] 네트워크 연결 끊김 시 에러 메시지
- [ ] 500 에러 시 서버 오류 메시지
- [ ] 403 에러 시 권한 없음 메시지
- [ ] 404 에러 시 리소스 없음 메시지
- [ ] 타임아웃 시 에러 메시지

**빈 상태 테스트**
- [ ] Slot 없음 메시지 표시
- [ ] 시나리오 없음 메시지 표시
- [ ] 검색 결과 없음 메시지 표시
- [ ] 필터 결과 없음 메시지 표시

**로딩 상태 테스트**
- [ ] 데이터 로딩 중 스피너 + 메시지 표시
- [ ] 로딩 완료 후 메시지 사라짐
