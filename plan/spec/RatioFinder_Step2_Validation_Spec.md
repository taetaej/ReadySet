# Ratio Finder Step2 — Validation 명세서

> 시나리오 생성 > Step2 상세 설정 (Ratio Finder) 화면의 유효성 검사 정책

---

## 1. 검사 대상 필드 총괄

| # | 필드 | 필수 | 입력 유형 | 검사 항목 |
|---|------|------|-----------|-----------|
| 1 | 총 예산 | ✅ | text (숫자) | 미입력, 0 이하 |
| 2 | 시뮬레이션 단위 | ✅ | 버튼 그룹 (5%/10%/20%) | 미선택 |
| 3 | 매체 선택 (DIGITAL) | ✅ | 체크박스 | 최소 1개 |
| 4 | 매체 선택 (TVC) | ✅ | 체크박스 | 최소 1개 |
| 5 | 매체 비중 (%) | ✅ | number input | 0% 또는 미입력 |
| 6 | 매체 비중 합계 | ✅ | 자동 계산 | DIGITAL 합계 = 100%, TVC 합계 = 100% |
| 7 | 상품/채널 선택 | ✅ | 다이얼로그 | 매체당 최소 1개 |
| 8 | 상품/채널 비중 (%) | ✅ | number input | 0% 또는 미입력 |
| 9 | 상품/채널 비중 합계 | ✅ | 자동 계산 | 매체별 합계 = 100% |

## 2. 필드별 상세 검사 규칙

### 2-1. 총 예산

| 항목 | 내용 |
|------|------|
| 검사 조건 | `!formData.totalBudget \|\| formData.totalBudget <= 0` |
| 표시 시점 | `validationActive === true` |
| border 변경 | input borderColor → `hsl(var(--destructive))` |
| 메시지 | "총 예산을 입력해주세요." |
| 메시지 위치 | input 아래, `marginTop: 4px` |
| 다음 버튼 | 비활성화 |

### 2-2. 시뮬레이션 단위

| 항목 | 내용 |
|------|------|
| 검사 조건 | `!formData.simulationUnit` |
| 표시 시점 | `validationActive === true` |
| border 변경 | 3개 버튼 모두 border → `hsl(var(--destructive))` |
| 메시지 | "시뮬레이션 단위를 선택해주세요." |
| 메시지 위치 | 버튼 그룹 아래, `marginTop: 8px` |
| 다음 버튼 | 비활성화 |

### 2-3. 매체 선택 (카테고리별 최소 1개)

| 항목 | 내용 |
|------|------|
| 검사 조건 | DIGITAL 매체 0개 또는 TVC 매체 0개 |
| 표시 시점 | `validationActive === true` |
| 메시지 | "DIGITAL 카테고리에서 최소 1개 이상의 매체를 선택해주세요." / "TVC 카테고리에서 최소 1개 이상의 매체를 선택해주세요." |
| 메시지 위치 | 매체별 예산 배분 영역 바깥 하단 (탭 컨테이너 밖), `marginTop: 12px` |
| 다음 버튼 | 비활성화 |

### 2-4. 매체 비중 0%

| 항목 | 내용 |
|------|------|
| 검사 조건 | `!mediaRatios[mediaKey] \|\| mediaRatios[mediaKey] === 0` |
| 표시 시점 | 항상 (매체가 선택된 상태이면 `validationActive` 무관) |
| 메시지 | "비중이 0%입니다. 값을 입력하거나 매체를 선택 해제해주세요." |
| 메시지 위치 | 해당 매체 행 바로 아래 (grid 밖), `padding: 4px 0 0 40px` |
| 다음 버튼 | 비활성화 |

> ⚠️ 핵심: 미입력(`undefined`)과 `0` 모두 동일하게 0%로 간주한다. `=== 0`만으로는 미입력을 잡지 못함.

### 2-5. 매체 비중 합계 ≠ 100%

| 항목 | 내용 |
|------|------|
| 검사 조건 | 탭별(DIGITAL/TVC) 선택된 매체 비중 합계 ≠ 100 |
| 표시 시점 | 항상 (해당 탭에 선택된 매체가 1개 이상이면) |
| 합계 숫자 색상 | `hsl(var(--destructive))` (빨간색) |
| 메시지 | "{DIGITAL\|TVC} 매체 비중 합계가 {N}%입니다. 100%로 맞춰주세요." |
| 메시지 위치 | 탭 내부 하단, 합계 표시 아래 |
| 다음 버튼 | 비활성화 |

### 2-6. 상품/채널 미선택

| 항목 | 내용 |
|------|------|
| 검사 조건 | 매체가 선택되었으나 상품/채널이 0개 |
| 표시 시점 | 항상 |
| UI 힌트 | dashed border 영역 + ListPlus 아이콘 + "'상품 추가' 버튼을 클릭하여 상품을 추가하세요" |
| 메시지 | "• {매체명}: 최소 1개 이상의 상품을 선택해주세요." |
| 메시지 위치 | 탭 내부 하단 (합계 영역 아래) |
| 다음 버튼 | 비활성화 |

### 2-7. 상품/채널 비중 0%

| 항목 | 내용 |
|------|------|
| 검사 조건 | `!productRatios[mediaKey]?.[product] \|\| productRatios[mediaKey]?.[product] === 0` |
| 표시 시점 | 항상 (상품이 추가된 상태이면) |
| 메시지 | "비중이 0%입니다. 값을 입력하거나 삭제해주세요." |
| 메시지 위치 | 해당 상품 행 내부, `gridColumn: '1 / -1'` |
| 다음 버튼 | 비활성화 |

> ⚠️ 매체 비중과 동일하게 미입력(`undefined`)과 `0` 모두 잡아야 함.

### 2-8. 상품/채널 비중 합계 ≠ 100%

| 항목 | 내용 |
|------|------|
| 검사 조건 | 매체 내 상품 비중 합계 ≠ 100 |
| 표시 시점 | 항상 (상품이 1개 이상이면) |
| 컨테이너 border | `hsl(var(--destructive))` (빨간색) |
| 합계 숫자 색상 | `hsl(var(--destructive))` |
| 메시지 | "비중 합계가 {N}%입니다. 100%로 맞춰주세요." |
| 메시지 위치 | 상품 목록 하단, 합계 표시 아래 |
| 다음 버튼 | 비활성화 |

## 3. "다음" 버튼 비활성화 조건 (isStep2Valid)

`CreateScenario.tsx`의 `isStep2Valid()` 함수에서 아래 조건을 순서대로 검사하며, 하나라도 실패하면 `false` 반환 → 다음 버튼 `disabled`.

```
1. 총 예산 미입력 또는 0 이하
2. 시뮬레이션 단위 미선택
3. DIGITAL 매체 0개 또는 TVC 매체 0개
4. 매체에 상품/채널이 0개인 항목 존재
5. 매체 비중이 0% 또는 미입력인 항목 존재     ← 신규 추가
6. 상품/채널 비중이 0% 또는 미입력인 항목 존재  ← 신규 추가
7. 매체별 상품 비중 합계 ≠ 100%
8. DIGITAL 매체 비중 합계 ≠ 100%
9. TVC 매체 비중 합계 ≠ 100%
```

## 5. border 색상 변경 규칙

| 대상 | 조건 | 정상 | 에러 |
|------|------|------|------|
| 총 예산 input | 미입력/0이하 + validationActive | `기본` | `hsl(var(--destructive))` |
| 시뮬레이션 단위 버튼 | 미선택 + validationActive | `hsl(var(--border))` | `hsl(var(--destructive))` |
| 상품 목록 컨테이너 | 비중 합계 ≠ 100% | `hsl(var(--border))` | `hsl(var(--destructive))` |

> 매체 비중 input, 상품 비중 input에는 border 변경 없음 (메시지로만 안내)

현재 구현 기준 (추후 플랫폼 정책에 맞춰 12px로 통일 예정):
