# Ratio Finder Step2 — 비율 자동 균등 분배 정책

## 배경 및 목적

Ratio Finder Step2에서 매체/상품을 선택하면 사용자가 직접 비율(%)을 입력해야 한다.
매체 3개, 상품 5개씩 선택하면 총 18개 입력 필드를 수동으로 채워야 하는 번거로움이 있었다.

이를 해소하기 위해 **선택/제거 시 자동으로 균등 분배 기본값을 세팅**하는 정책을 도입한다.

---

## 핵심 규칙

| 구분 | 합계 기준 |
|---|---|
| DIGITAL 카테고리 내 매체끼리 | 합 100% |
| 매체 내 상품끼리 | 합 100% |
| TV 카테고리 내 매체끼리 | 합 100% |
| 매체 내 채널끼리 | 합 100% |

DIGITAL과 TV는 독립적으로 각각 100%를 유지한다.

---

## 분배 공식

```
base = Math.floor(100 / n)
remainder = 100 - base * n
각 항목 = base + (앞에서부터 remainder개만 +1)
```

### 예시

| 항목 수 | 분배 결과 |
|---|---|
| 1개 | 100 |
| 2개 | 50, 50 |
| 3개 | 34, 33, 33 |
| 4개 | 25, 25, 25, 25 |
| 5개 | 20, 20, 20, 20, 20 |
| 7개 | 15, 15, 14, 14, 14, 14, 14 |

나머지(remainder)는 앞쪽 항목부터 +1씩 부여하여 합이 정확히 100이 되도록 보장한다.

## 동작 케이스 — 매체 비율

### Case 1: 매체 추가

| 시나리오 | 동작 |
|---|---|
| DIGITAL에서 Google Ads 선택 (첫 매체) | Google Ads = 100% |
| 이어서 Meta 선택 | Google Ads = 50%, Meta = 50% |
| 이어서 TikTok 선택 | Google Ads = 34%, Meta = 33%, TikTok = 33% |

**규칙**: 매체가 추가될 때마다 같은 카테고리 내 전체 매체를 균등 재분배한다.

### Case 2: 매체 제거

| 시나리오 | 동작 |
|---|---|
| Google 34%, Meta 33%, TikTok 33% 상태에서 TikTok 해제 | Google = 50%, Meta = 50% |
| Google 50%, Meta 50% 상태에서 Meta 해제 | Google = 100% |

**규칙**: 매체가 제거되면 남은 매체들끼리 균등 재분배한다.

### Case 3: 사용자가 직접 수정 후 매체 추가

| 시나리오 | 동작 |
|---|---|
| Google 60%, Meta 40%으로 수정한 상태에서 TikTok 추가 | Google = 34%, Meta = 33%, TikTok = 33% (리셋) |

**규칙**: 사용자가 수정한 값이 있더라도 매체 추가/제거 시 항상 균등 재분배된다.

---

## 동작 케이스 — 상품/채널 비율

### Case 4: 상품 선택 (다이얼로그 확인)

| 시나리오 | 동작 |
|---|---|
| Google Ads에서 상품 2개 선택 후 확인 | 상품A = 50%, 상품B = 50% |
| Google Ads에서 상품 3개 선택 후 확인 | 상품A = 34%, 상품B = 33%, 상품C = 33% |

**규칙**: 상품 선택 다이얼로그에서 확인을 누르면 선택된 상품들을 항상 균등 분배한다.

### Case 5: 상품 제거 (X 버튼)

| 시나리오 | 동작 |
|---|---|
| 상품A 34%, 상품B 33%, 상품C 33% 상태에서 상품C 제거 | 상품A = 50%, 상품B = 50% |
| 상품A 60%, 상품B 40% 수정 후 상품B 제거 | 상품A = 100% |

**규칙**: 상품이 제거되면 남은 상품들끼리 균등 재분배한다.

### Case 6: 상품 재선택 (다이얼로그 다시 열어서 변경)

| 시나리오 | 동작 |
|---|---|
| 기존 상품A 60%, 상품B 40% → 다이얼로그에서 상품C 추가 후 확인 | 상품A = 34%, 상품B = 33%, 상품C = 33% (리셋) |

**규칙**: 다이얼로그 확인 시 항상 전체 균등 재분배. 기존 수정값은 보존하지 않는다.

## 설계 결정 사유

### "왜 값을 직접 수정할 때 나머지를 자동 재분배하지 않는가?"

균등 분배된 상태(예: 34, 33, 33)에서 사용자가 하나를 60으로 수정했을 때, 나머지를 자동으로 20, 20으로 맞춰주는 방식을 검토했으나 채택하지 않았다.

**이유**: 1개를 바꿀 때마다 나머지 항목들의 값이 계속 바뀌면, 사용자가 원하는 값으로 세팅하기 어렵다. 예를 들어 A=60, B=30, C=10을 의도했는데, A를 60으로 바꾸는 순간 B와 C가 20, 20이 되고, B를 30으로 바꾸면 C가 10이 되긴 하지만 — 항목이 5개 이상일 때는 하나 수정할 때마다 나머지 4개가 연쇄적으로 바뀌어 혼란스럽다.

따라서 **값 직접 수정 시에는 해당 항목만 변경되고, 합이 100%가 아니면 검증 UI가 안내**하는 방식을 채택했다.

### "왜 항상 균등 재분배인가? 기존 값을 보존하지 않는 이유"

1. **순차 선택 문제**: 매체는 하나씩 체크하므로, 첫 매체가 100%가 되고 이후 추가 매체가 0%가 되는 비직관적 상황이 발생한다.
2. **고정-잔여 방식의 복잡성**: "사용자가 수정한 값은 고정, 나머지만 재분배"는 추적 상태가 필요하고, 항목이 많을 때 소수점 단위로 값이 바뀌어 오히려 혼란스럽다.
3. **사용 플로우와의 정합성**: 실제 사용 패턴은 "매체를 쭉 선택 → 비율 조정"이므로, 선택 단계에서는 균등 분배가 자연스럽다.
4. **구현 단순성**: 상태 추적 없이 `Math.floor(100/n)` + 나머지 보정만으로 구현 가능.

### "왜 DIGITAL과 TV를 독립적으로 100%씩?"

Ratio Finder는 카테고리별로 독립적인 예산 배분 최적화를 수행한다. DIGITAL 내에서의 비율과 TV 내에서의 비율은 별개의 시뮬레이션 축이다.

---

## 검증 (기존 유지)

- 각 카테고리 내 매체 비율 합 ≠ 100% → 경고 메시지 표시
- 각 매체 내 상품 비율 합 ≠ 100% → 경고 메시지 표시
- 비율 0%인 항목 → 개별 경고 표시

사용자가 균등 분배 후 직접 수정하여 합이 100%가 아닌 상태가 되면, 기존 검증 UI가 안내한다.

## 상품/채널 선택 다이얼로그 UI 스펙

### 구조

```
┌─────────────────────────────────────────┐
│ {매체명} 상품 선택                        │  ← 타이틀 (bold, 18px)
│ 분석에 포함할 상품을 선택하세요             │  ← 설명 (muted, 14px)
│                                         │
│ ┌─────────────────────────────────────┐ │
│ │ ⓘ Step1에서 선택한 업종의 모델 학습   │ │  ← 안내 배너 (info)
│ │   결과가 있는 광고상품만 표시됩니다.   │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ ┌─────────────────────────────────────┐ │
│ │ 🔍 검색...                          │ │  ← 검색 입력
│ └─────────────────────────────────────┘ │
│                                         │
│ ┌─────────────────────────────────────┐ │
│ │ ☐ 일반 디스플레이 광고_CPC           │ │  ← 체크박스 리스트
│ ├─────────────────────────────────────┤ │
│ │ ☐ 반응형 디스플레이 광고_CPC         │ │
│ ├─────────────────────────────────────┤ │
│ │ ☐ 디맨드젠 동영상 광고_CPC          │ │
│ ├─────────────────────────────────────┤ │
│ │ ☐ ...                              │ │
│ └─────────────────────────────────────┘ │
│                                         │
│              [취소]  [확인 (N개 선택)]    │  ← 하단 버튼
└─────────────────────────────────────────┘
```

### 상세 스펙

| 항목 | 스펙 |
|---|---|
| 타이틀 | "{매체명} 상품 선택" (DIGITAL) / "{매체명} 채널 선택" (TV) |
| 설명 | "분석에 포함할 상품을 선택하세요" / "분석에 포함할 채널을 선택하세요" |
| 안내 배너 | Info 아이콘 + "Step1에서 선택한 업종의 모델 학습 결과가 있는 광고상품만 표시됩니다." |
| 검색 | 상품명 필터링 (대소문자 무시) |
| 리스트 아이템 | 체크박스 + 상품명, 카드형 (border, rounded, padding 12px) |
| 확인 버튼 | "확인 (N개 선택)" — 선택 개수 실시간 반영, primary 스타일 (검정 배경, 흰 텍스트) |
| 취소 버튼 | secondary 스타일 |
| 스크롤 | 리스트 영역 스크롤, 다이얼로그 max-height 90vh |

### 동작

1. 다이얼로그 열기: 해당 매체에 이미 선택된 상품이 있으면 체크 상태로 표시
2. 검색: 입력값으로 상품명 필터링
3. 확인: 선택된 상품들에 대해 균등 분배 적용 후 다이얼로그 닫기
4. 취소: 변경 없이 닫기
5. 오버레이 클릭: 취소와 동일
