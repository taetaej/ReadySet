### 2. import 작성 시

### 3. 스타일 작성 시
```typescript

  background: 'hsl(var(--background))',

}}>

// ❌ 나쁜 예 (하드코딩)

  background: '#ffffff',  // 다크모드에서 안 보임!
```

## 🚨 에러 발생 시 대응

### 1. "Cannot find module" 에러
```bash
# 해결 방법
npm install
```

### 2. "Unexpected token" 에러
```typescript
// 원인: import 경로 오류
// 해결: 경로 확인 및 수정
import { Avatar } from '../common/Avatar'  // ✅
```

### 3. 화면이 안 나올 때
```typescript
// 1. export 확인
export function DataShotList() { ... }  // ✅

// 2. import 확인
import { DataShotList } from './DataShotList'  // ✅

// 3. 라우팅 확인 (App.tsx)
<Route path="/datashot" element={<DataShotDetail />} />
```

### 4. 스타일이 안 먹힐 때
```typescript
// 원인: CSS 변수 미사용
// 해결: hsl(var(--변수명)) 사용
background: 'hsl(var(--background))'  // ✅
```

## 📦 Git 작업 가이드

### 안전한 커밋 방법

```bash
# 1. 상태 확인
git status

# 2. datashot 폴더만 추가
git add src/components/datashot/

# 3. 커밋 메시지 작성
git commit -m "DataShot: 시나리오 목록 구현"

# 4. 푸시 (필요시)
git push
```

### 커밋 메시지 규칙

```
DataShot: [작업 내용]

예시:
- DataShot: 시나리오 목록 컴포넌트 추가
- DataShot: 시나리오 카드 UI 구현
- DataShot: 검색 기능 추가
- DataShot: 버그 수정 - 목록 정렬 오류
```

### 실수로 다른 파일을 커밋했다면?

```bash
# 마지막 커밋 취소 (파일은 유지)
git reset --soft HEAD~1

# 다시 datashot 폴더만 추가
git add src/components/datashot/
git commit -m "DataShot: 올바른 커밋"
```
